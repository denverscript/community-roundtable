* MCP
  * What is MCP?
    * A standardized way to give AI agents access to tools.
    * https://modelcontextprotocol.io/docs/getting-started/intro
  * What can tools do?
    * Anything that code can do.
      * Run a program
      * Call Web APIs
      * Access the file system
      * Read from a database
      * Send a message
  * Favorite MCP Servers for development
    * [Context7](https://context7.com/)
      * Give your agents direct access to developer documentation. They fully index docs so agents can semantically search for specific docs without bloating the context.
    * [Playwright](https://github.com/microsoft/playwright-mcp)
      * Give your agents direct access to a web browser. They can do anything listed in the tools section [here](https://github.com/microsoft/playwright-mcp#tools).
  * MCP vs CLI
    * When you enable an MCP server, the list of tools is added to the context of _every_ request. This can increase cost / token usage. Many people think just asking an Agent to use a CLI tool is a better use of tokens
      * For example use the [agent-browser](https://github.com/vercel-labs/agent-browser) instead of the Playwright MCP server
      * See a discussion about it on Reddit [here](https://www.reddit.com/r/startups/comments/1rv8tcx/mcp_vs_cli_whats_your_take_on_the_discussion_in/)
* Skills

    * Markdown files that live in your repo that describe specific capabilities, expertise or workflows
  * There are many skills directories, BE CAREFUL with which you use and what you install. The markdown contained in the skills can have _anything_ so malicious folks publish skills with prompt injections etc.
    * This directory is hosted by Vercel https://www.skills.sh/
      * ANYONE can publish a skill. Verify / validate the skill yourself first before installing anything

* Agent Hooks
  * What are hooks?
    * Intercept agent behavior. Tap into before / after tool calls to guide the agent.
  * When might you use this?
    * Tap into _before_ tool call - stop an agent from running "npm" in a "pnpm" project
    * Tap into _after_ tool call - write a log entry to a file
  * Most agents support Hooks
    * [Claude](https://code.claude.com/docs/en/hooks)
    * [Co-Pilot](https://docs.github.com/en/copilot/reference/hooks-reference)
    * [VS Code](https://code.visualstudio.com/docs/agent-customization/hooks)
* Spec Driven Development
  * What is it?
    * Spec is short for Specification
      * Specifications are documents that describe architectures, features and functionality
    * Essentially Plan mode on steroids. Spec tools guide the agent to create PRDs (Product Requirements Documents). These specs then live in the code base. Agents use these specs when working on features.
    * The tools to work this way take various forms:
      * CLI tools that guide AI to generate specification docs
        * You then use your preferred AI coding tool to work with these generated docs
      * Entire IDEs built around a spec driven flow
    * See [What are Specs?](https://kiro.dev/docs/specs/)
    * See [Understanding Spec-Driven-Development: Kiro, spec-kit, and Tessl](https://martinfowler.com/articles/exploring-gen-ai/sdd-3-tools.html)
  * Tools
    * [spec-kit](https://github.com/github/spec-kit)
    * [Kiro](https://kiro.dev)

* Deep Research
  * What is it?
    * A way to set AI off on a research task. It will run in the background and produce a report for you.
  * How does it work?
    * Typically a "harness" or "wrapper" around an existing model, provided by a service.
    * A form of "agent" that is set off on a specific task.
    * It uses web search and web page retrieval tools to search, read and summarize the web.
    * It then takes all of the data retrieved and formats it into a formalized report.
  * What can you use it for?
    * Product research - "I'm looking for a laptop with 16GB of RAM and 12GB of VRAM with at least 8 cores, what are my best options?"
    * Market research - "I want to start a company that sells socks for dogs. What companies already exist? What is the market size? What interest is there for this"
    * Development research - "I want to build a front-end app with these features... what are my best options for this stack? What example apps and architecture exist similar to this on github?"
  * Supported By Many Tools
    * [ChatGPT](https://openai.com/index/introducing-deep-research/)
    * [Gemini](https://gemini.google/us/overview/deep-research/)
    * [Claude](https://claude.com/blog/research)
* Agent Goal / Loops
  * What is it?
    * Set AI off on a task with a specific end goal
    * The AI will keep iterating until the goal is met
  * Example
    * "Given the this test suite, do not stop until all tests pass. Do not modify existing tests or add new tests."
    * "Work through the list of tasks in tasks.md Do not stop until all tasks are complete."
  * Goals are supported by many tools directly or as community plugins
    * [Claude](https://code.claude.com/docs/en/goal)
* Prompt Tips
  * Sometimes restarting a session with the same prompt but slightly modified is OKAY. Saves on tokens / iterating and resets to a specific goal / clarification.
* Context Management
  * Every model has a context limit measured in tokens.
  * Your entire chat history is sent on every request.
  * Most tools have a way of seeing how much context is taken up / left
  * Keep an eye on used context used!
  * If a chat history reaches the context limit, many tools have a built in "summary" mode, which will summarize the context BUT this can sometimes miss small details from the history. Avoid reaching summary mode as much as possible
* How to keep up to date? What to Use?
  * Be Pragmatic
    * Pick one model / provider - the latest models from the major providers like GPT 5.5 and Opus 4.8 are _very_ similar in capability. Pick one and use that.
    * If you use a service like Github Copilot - you get access to multiple providers.
* Don't Just Plug anything into an AI prompt box
  * Every prompt you type is sent to an AI companies servers
  * AI companies have various policies about how they use the prompts you send to them
    * Enterprise (We will not train on your data...)
      * If you work at a company with IP, Secrets, Health Data etc. you will need to look into enterprise plans that support this
        * OpenAI
        * Anthropic
    * Personal
      * Check the box in settings (do not train on my data)

# What models exist? Who makes them? Which should I use?

It costs billions of dollars to create a model, so we mainly only see the highest capability models from large companies and heavily funded startups. Open weight models (typically Chinese companies) are getting better and smaller, but the biggest / latest models from the big companies are still hard to beat.

When you are using an AI coding tool, you are typically going to choose one of these big providers / models. Each model is a certain size (bigger is slower / smarter / more expensive, smaller is faster / dumber / cheaper) and each might support various "thinking" modes.

When a new model or version comes out from one provider, another provider is usually quick to launch theirs as well. Typically we reach for the biggest and latest model from any given provider when coding. These are all very similar in capability. Right now the latest best models would be:
* Anthropic: Claude Opus 4.6+
* ChatGPT: 5.4 Codex / 5.5
* Google: Gemini 3 Pro

See [SWE Bench](https://www.swebench.com/) for benchmarks of current models.
